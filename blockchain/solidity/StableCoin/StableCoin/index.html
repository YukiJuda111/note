    类似于DAI的stable coin构建

    记录了我构建类似于DAI的stable coin的全流程，参考了DAI的实现。

    1 稳定币概述

    项目中的稳定币是参考了DAI，1:1与美元挂钩的稳定币。用了liquidate的算法来保证1:1的稳定性。其中liquidate算法也是该项目中的重点，即当用户的质押物不足以维持其取出的stable coin的数量时(低于health factor)，则其他用户可以对该用户进行清算，即通过较便宜的价格购买低于health factor用户的质押物，以此来维持整个DeFi始终保持over-collateralized(质押物的价值 > 已经铸造的稳定币的价值)。通过维持整个DeFi的over-collateralized的性质，来维持1:1的稳定性。liquidate函数会在后续展开介绍。

    2 构建ERC20

    代码：https://github.com/YukiJuda111/StableCoin/blob/main/src/DecentralizedStableCoin.sol

    用了openzepplin库来快速构建ERC20，稳定币主要功能是提供mint和burn接口，供DscEngine使用。后续称该ERC20 token为DSC。

    • mint: 在逻辑上负责铸造token，_mint调用负责将token从0x0发送到指定地址(user)
    function mint(address _to, uint256 _amount) external onlyOwner returns (bool) {
    if (_to == address(0)) {
        revert DecentralizedStableCoin__NotZeroAddress();
    }
    if (_amount <= 0) {
        revert DecentralizedStableCoin__MustBeMoreThanZero();
    }
    _mint(_to, _amount);
    return true;
}
    • burn: 在逻辑上负责销毁token，_burn调用本质上讲msg.sender的token还给0x0
    function mint(address _to, uint256 _amount) external onlyOwner returns (bool) {
    if (_to == address(0)) {
        revert DecentralizedStableCoin__NotZeroAddress();
    }
    if (_amount <= 0) {
        revert DecentralizedStableCoin__MustBeMoreThanZero();
    }
    _mint(_to, _amount);
    return true;
}

    3 构建DscEngine(Defi)

    代码：https://github.com/YukiJuda111/StableCoin/blob/main/src/DSCEngine.sol

    3.1 构造函数

    需要传入:

    • 作为质押物的token地址(weth和wbtc的地址)
    • 喂价地址: 用于换算质押物token和美元的汇率
    • dsc地址(刚刚构建的ERC20的地址)
    constructor(address[] memory tokenAddresses, address[] memory priceFeedAddresses, address dscAddress) {
    if (tokenAddresses.length != priceFeedAddresses.length) {
        revert DSCEngine__TokenAddressAndPriceFeedAddressMustBeEqualLength();
    }
    for (uint256 i = 0; i < tokenAddresses.length; i++) {
        s_priceFeeds[tokenAddresses[i]] = priceFeedAddresses[i];
        s_collateralTokens.push(tokenAddresses[i]);
    }
    i_dsc = DecentralizedStableCoin(dscAddress);
}

    3.2 HealthFactor

    检测用户的health factor是整个稳定币DeFi能运作的核心。当用户的的health factor低于一定阈值时，我们就要让其他用户对其进行liquidate。

    health factor具体而言就是用户铸造的DSC和质押进去的token的比值，在这个合约中设定为1:2来满足整个Defi始终保持over-collateralized，例如质押$200价值的token则最多可以在defi中铸造$100价值的DSC。代码如下：

    function _healthFactor(address user) private view returns (uint256) {
    // total DSC minted
    // total collateral deposited
    (uint256 totalDscMinted, uint256 totalCollateralInUsd) = _getAccountInformation(user);
    if (totalDscMinted == 0) return type(uint256).max; // wtf
    uint256 collateralAdjustedForThreshold = totalCollateralInUsd * LIQUIDATION_THRESHOLD / LIQUIDATION_PRECISION;
    // 200% over-collateralized
    return (collateralAdjustedForThreshold * PRECISON) / totalDscMinted;
}

    为了计算上面代码中的totalCollateralInUsd，需要使用chainlink的预言机:https://docs.chain.link/data-feeds/price-feeds/addresses?network=ethereum&page=1

    function getUsdValue(address token, uint256 amount) public view returns(uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(s_priceFeeds[token]);
    ( , int price, , , ) = priceFeed.stablePriceCheck();
    // 从chainlink返回的价格是8位小数
    return uint256(price) * ADDITIONAL_FEED_PRECISION * amount / PRECISON;
}

    还要注意if (totalDscMinted == 0) return type(uint256).max;。即用户只选择存入质押物不去铸造DSC这当然也是可行的，要特殊判断。这是通过后续fuzz test找到的bug，会在后面再次提到。

    3.2 存入质押物并给msg.sender铸造DSC

    3.2.1 存入质押物

    /**
 * @param tokenCollateral 作为抵押物的token的地址
 * @param amoutCollateral 抵押物的数量
 */
function depositCollateral(address tokenCollateral, uint256 amoutCollateral)
    public
    moreThanZero(amoutCollateral)
    isAllowedToken(tokenCollateral)
    nonReentrant // 不可重入(more gas fee, safer)
{
    s_collateralDeposited[msg.sender][tokenCollateral] += amoutCollateral;
    emit CollateralDeposited(msg.sender, tokenCollateral, amoutCollateral);
    bool success = IERC20(tokenCollateral).transferFrom(msg.sender, address(this), amoutCollateral);
    if(!success) {
        revert DSCEngine__TransferFailed();
    }
    _revertIfHealthFactorIsBroken(msg.sender);
    // console.log("now balance: ", IERC20(tokenCollateral).balanceOf(address(this)));
}
