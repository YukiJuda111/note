    depositAndMint

    DscEngine 质押并铸造稳定币

    目标

    类似与DAI的stablecoin：

    1. Relative Stability: Pegged -> 1USD
    2. Chainlink PriceFeed
    3. Set a function to exchange ETH & BTC -> USD
    4. Stability Mechanism: Algorithmic (Decentralized)
    5. People can only mint the stablecoin with enough collateral
    6. Collateral: Exogenous (Crypto)
    7. wETH
    8. wBTC

    DscEngine

    Defi的主要逻辑由DscEngin.sol实现：

    • 构造函数
    • 质押

    构造函数

    在部署合约时，由外部提供质押代币地址，喂价地址和token地址(需要先部署token再部署engine)。质押的代币采用:weth,wbtc。

    constructor(address[] memory tokenAddresses, address[] memory priceFeedAddresses, address dscAddress) {
    if (tokenAddresses.length != priceFeedAddresses.length) {
        revert DSCEngine__TokenAddressAndPriceFeedAddressMustBeEqualLength();
    }
    for (uint256 i = 0; i < tokenAddresses.length; i++) {
        s_priceFeeds[tokenAddresses[i]] = priceFeedAddresses[i];
        s_collateralTokens.push(tokenAddresses[i]);
    }
    i_dsc = DecentralizedStableCoin(dscAddress);
}

    质押

    传入质押代币的地址，因为weth和wbtc采用ERC20标准，可以通过IERC20的接口发送函数签名去调用。同时用s_collateralDeposited去记录每个用户的质押代币数量。

    mapping(address user => mapping(address token => uint256 amount)) private s_collateralDeposited;

    function depositCollateral(address tokenCollateral, uint256 amoutCollateral)
    external
    moreThanZero(amoutCollateral)
    isAllowedToken(tokenCollateral)
    nonReentrant // 不可重入(more gas fee, safer)
{
    s_collateralDeposited[msg.sender][tokenCollateral] += amoutCollateral;
    emit CollateralDeposited(msg.sender, tokenCollateral, amoutCollateral);
    bool success = IERC20(tokenCollateral).transferFrom(msg.sender, address(this), amoutCollateral);
    if(!success) {
        revert DSCEngine__TransferFailed();
    }
}

    铸造DSC

    mintDsc

    调用刚才自己编写的ERC20 Token的mint来铸造DSC。

    function mintDsc(uint256 amountDscToMint)
    external 
    moreThanZero(amountDscToMint)
    nonReentrant
{
    s_dscMinted[msg.sender] += amountDscToMint;
    // if minted too much, revert(e.g. $150 DSC, $100 collateral)
    _revertIfHealthFactorIsBroken(msg.sender);
    bool minted = i_dsc.mint(msg.sender, amountDscToMint);
    if(!minted){
        revert DSCEngine__MintFailed();
    }
}

    这里有一层逻辑判断，要保证用户的HealthFactor是正常的，即确保用户没有仅存入一点质押物却铸造很多DSC的情况，本文中的HealthFactor是200%，即铸造$100 DSC至少需要$200 质押物。

    _revertIfHealthFactorIsBroken

    在mintDsc中用到的函数

    function _revertIfHealthFactorIsBroken(address user) internal view {
    uint256 userHealthFactor = _healthFactor(user);
    if (userHealthFactor < MIN_HEALTH_FACTOR) {
        revert DSCEngine__BreaksHealthFactor(userHealthFactor);
    }
}

    _healthFactor

    在判断healthFactor时，对比用户的dsc和质押物的价值即可(200%)。这里设计小数计算，在solidity中需要注意一下。

    function _healthFactor(address user) private view returns (uint256) {
    // total DSC minted
    // total collateral deposited
    (uint256 totalDscMinted, uint256 totalCollateralInUsd) = _getAccountInformation(user);
    uint256 collateralAdjustedForThreshold = totalCollateralInUsd * LIQUIDATION_THRESHOLD / LIQUIDATION_PRECISION;
    // 200% over-collateralized
    return (collateralAdjustedForThreshold * PRECISON) / totalDscMinted;
}

    为了获得质押物的价值，我们要用上pricefeed。

    priceFeed

    省略了部分从healthfactor到pricefeed的逻辑，重点是使用了chainlink的AggregatorV3的接口，给出了某一token的pricefeed地址，调用latestRoundData来获取price。从chainlink返回的price是8位小数，也要注意小数计算。

    function getUsdValue(address token, uint256 amount) public view returns(uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(s_priceFeeds[token]);
    ( , int price, , , ) = priceFeed.latestRoundData();
    // 从chainlink返回的价格是8位小数
    return uint256(price) * ADDITIONAL_FEED_PRECISION * amount / PRECISON;
}
